6.1 Evaluar Modelos con Conjuntos de Entrenamiento y Prueba

El mtodo ms simple que podemos utilizar para evaluar el rendimiento de un algoritmo de aprendizaje automtico es usar diferentes conjuntos de datos de entrenamiento y prueba. Podemos tomar nuestro conjunto de datos original y dividirlo en dos partes. Entrenamos el algoritmo en la primera parte, luego hacemos predicciones en la segunda parte y evaluamos las predicciones comparndolas con los resultados esperados. El tamao de la divisin puede depender del tamao y las especificaciones de tu conjunto de datos, aunque es comn utilizar el 67% de los datos para el entrenamiento y el 33% restante para las pruebas.

Esta tcnica de evaluacin de algoritmos es rpida. Es ideal para conjuntos de datos grandes (millones de registros) donde hay una fuerte evidencia de que ambas divisiones de los datos son representativas del problema subyacente. Debido a la velocidad, es til usar este enfoque cuando el algoritmo que ests investigando es lento de entrenar. Un inconveniente de esta tcnica es que puede tener una alta varianza. Esto significa que las diferencias en los conjuntos de datos de entrenamiento y prueba pueden resultar en diferencias significativas en la estimacin de la precisin del modelo. Podemos dividir el conjunto de datos en un conjunto de entrenamiento y prueba utilizando la funcin train_test_split() de la biblioteca scikit-learn. Por ejemplo, podemos dividir el conjunto de datos en un 67% para entrenamiento y un 33% para prueba de la siguiente manera:

# split data into train and test sets
X_train, X_test, y_train, y_test = train_test_split(X, Y, test_size=0.33, random_state=7)

6.2 Evaluar Modelos con Validacin Cruzada k-Fold

La validacin cruzada es un enfoque que puedes usar para estimar el rendimiento de un algoritmo de aprendizaje automtico con menos varianza que una nica divisin en conjuntos de entrenamiento y prueba. Funciona dividiendo el conjunto de datos en k partes (por ejemplo, k = 5 o k = 10). Cada divisin del conjunto de datos se llama pliegue (fold). El algoritmo se entrena en k 1 pliegues, manteniendo uno de reserva y probando en el pliegue reservado. Esto se repite de manera que cada pliegue del conjunto de datos tenga la oportunidad de ser el conjunto de prueba reservado.

Despus de realizar la validacin cruzada, obtienes k puntuaciones de rendimiento diferentes que puedes resumir usando una media y una desviacin estndar. El resultado es una estimacin ms fiable del rendimiento del algoritmo en nuevos datos dado tu conjunto de datos de prueba. Es ms preciso porque el algoritmo se entrena y evala mltiples veces en diferentes datos. La eleccin de k debe permitir que el tamao de cada particin de prueba sea lo suficientemente grande como para ser una muestra razonable del problema, al tiempo que permite suficientes repeticiones de la evaluacin de entrenamiento-prueba del algoritmo para proporcionar una estimacin justa del rendimiento del algoritmo en datos no vistos.

Para conjuntos de datos de tamao moderado, con miles o decenas de miles de observaciones, son comunes los valores de k de 3, 5 y 10. Podemos usar el soporte de validacin cruzada k-fold proporcionado en scikit-learn. Primero, debemos crear el objeto KFold especificando el nmero de pliegues y el tamao del conjunto de datos. Luego, podemos usar este esquema con el conjunto de datos especfico. La funcin cross_val_score() de scikit-learn nos permite evaluar un modelo utilizando el esquema de validacin cruzada y devuelve una lista de las puntuaciones para cada modelo entrenado en cada pliegue.

kfold = KFold(n_splits=10, random_state=7)
    results = cross_val_score(model, X, Y, cv=kfold)
